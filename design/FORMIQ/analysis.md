# Analysis

## Purpose

Analysis tools should make spatial decisions understandable without turning the interface into a dashboard wall.

## Layout

- Keep the map visible.
- Show results as overlays, legends, and concise panels.
- Use one focused analysis panel at a time when possible.
- Avoid stacked analytics cards unless comparing repeated metrics.

## Visual Hierarchy

- Primary result: large and direct
- Supporting metrics: compact captions
- Controls: grouped by workflow step
- Warnings and errors: clear but not oversized

## State Colors

- Success: valid result or completed analysis
- Warning: partial data, slow source, missing optional layer
- Danger: failed analysis or invalid geometry

## UX

- Analysis should feel like a professional GIS workflow.
- Explain uncertainty through compact labels.
- Do not expose internal provider complexity as primary UI.
